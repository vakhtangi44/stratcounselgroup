# 17 — API Routes

All API routes live in `src/app/api/`. Next.js App Router uses file-based routing — a `route.ts` file in `api/foo/` becomes the handler for `GET/POST /api/foo`.

---

## Auth Pattern (used in every admin route)

Every admin route starts with:
```typescript
const session = await auth()
if (!session) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
```
- `auth()` from `@/lib/auth` — reads the JWT session cookie
- Returns `null` if missing or invalid → 401 Unauthorized
- HTTP 401 means "you need to be authenticated"

## Public Routes

### `POST /api/contact`

**File:** `src/app/api/contact/route.ts`

```typescript
async function verifyTurnstile(token: string): Promise<boolean> {
  const secret = process.env.CLOUDFLARE_TURNSTILE_SECRET_KEY
  if (!secret) return true  // skip verification in development
  const res = await fetch('https://challenges.cloudflare.com/turnstile/v0/siteverify', {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: new URLSearchParams({ secret, response: token }),
  })
  const data = await res.json() as { success: boolean }
  return data.success
}
```
- Calls Cloudflare's server-side API to verify the CAPTCHA token
- The `secret` key proves this is a legitimate server (not a bot re-using a token)
- If `CLOUDFLARE_TURNSTILE_SECRET_KEY` is not set (dev environment), skip verification
- `application/x-www-form-urlencoded` — Cloudflare's required content type

```typescript
export async function POST(request: Request) {
  const body = await request.json()
  const { name, email, phone, subject, message, turnstileToken } = body

  if (!name || !email || !subject || !message) {
    return NextResponse.json({ error: 'Missing required fields' }, { status: 400 })
  }
  if (!turnstileToken || !(await verifyTurnstile(turnstileToken))) {
    return NextResponse.json({ error: 'CAPTCHA verification failed' }, { status: 400 })
  }

  await db.contactMessage.create({
    data: { name, email, phone: phone || null, subject, message },
  })
  return NextResponse.json({ ok: true })
}
```
- Validates required fields first, then CAPTCHA
- Saves to DB regardless of email success (message won't be lost if Gmail is down)
- Note: `sendContactEmail()` is NOT called here — the contact form only saves to the DB. The admin reads messages from the admin panel. (The email sending was likely removed or not yet connected.)

### `POST /api/newsletter/subscribe`

```typescript
export async function POST(req: Request) {
  const { email } = await req.json()
  if (!email) return NextResponse.json({ error: 'Email required' }, { status: 400 })

  await db.newsletterSubscriber.upsert({
    where: { email },
    create: { email },
    update: { active: true },  // reactivate if previously unsubscribed
  })
  return NextResponse.json({ ok: true })
}
```
- `upsert` = INSERT or UPDATE — if the email exists, updates it; if new, creates it
- `update: { active: true }` — if someone previously unsubscribed and re-subscribes, they get reactivated
- `unsubscribeToken` is auto-generated by PostgreSQL `uuid()` on create

### `POST /api/newsletter/unsubscribe`

```typescript
export async function POST(req: Request) {
  const { token } = await req.json()
  await db.newsletterSubscriber.update({
    where: { unsubscribeToken: token },
    data: { active: false },
  })
  return NextResponse.json({ ok: true })
}
```
- Token-based unsubscribe — no email needed
- Sets `active: false` (doesn't delete — preserves email for re-subscribe)

## Admin Routes

### `GET/POST /api/admin/blog`

**File:** `src/app/api/admin/blog/route.ts`

**GET — list all posts:**
```typescript
const posts = await db.blogPost.findMany({
  orderBy: { createdAt: 'desc' },
  include: { author: { select: { nameEn: true } }, tags: true },
})
```
- Includes the author's English name for display in the blog list table
- Includes all tags so the admin can see which practice areas each post is tagged with

**POST — create post:**
```typescript
const post = await db.blogPost.create({
  data: {
    slug, titleKa, titleEn,
    contentKa: sanitizeHtml(contentKa || ''),
    contentEn: sanitizeHtml(contentEn || ''),
    ...
    publishedAt: status === 'published' ? new Date() : null,
    tags: {
      create: (tags || []).map((practiceArea) => ({ practiceArea })),
    },
  },
})
```
- `sanitizeHtml()` is called on both content fields before saving
- `publishedAt` set to now if status is published, else null
- Tags created in the same transaction using nested `create`

### `GET/PUT/DELETE /api/admin/blog/[id]`

**File:** `src/app/api/admin/blog/[id]/route.ts`

```typescript
function parseId(id: string) {
  const n = parseInt(id)
  return isNaN(n) ? null : n
}
```
- URL parameters are always strings. `parseInt` converts to number.
- If the string is not a valid number, returns `null` → 400 Bad Request
- Prevents passing `NaN` to Prisma which would cause an unintelligible error

**PUT — update post:**
```typescript
tags: {
  deleteMany: {},      // delete all existing tags for this post
  create: tags.map(practiceArea => ({ practiceArea })),  // create fresh set
},
```
- Tags are replaced entirely on each update (not diffed)
- `deleteMany: {}` — deletes all tags where `postId = id` (the `{}` means "no additional filter")
- Then creates the new set
- This is simpler than a diff-and-patch approach and correct for small sets

### `GET/PUT /api/admin/settings`

**File:** `src/app/api/admin/settings/route.ts`

**GET — grouped settings:**
```typescript
const settings = await db.siteSetting.findMany({ orderBy: { id: 'asc' } })
const grouped: Record<string, typeof settings> = {}
for (const s of settings) {
  if (!grouped[s.category]) grouped[s.category] = []
  grouped[s.category].push(s)
}
return NextResponse.json(grouped)
```
- Groups settings by `category` field for the admin UI tabs
- Returns: `{ "hero": [...], "contact": [...], "footer": [...] }`

**PUT — batch update:**
```typescript
const body: { id: number; valueKa: string; valueEn: string }[] = await req.json()
for (const item of body) {
  await db.siteSetting.update({
    where: { id: item.id },
    data: { valueKa: item.valueKa, valueEn: item.valueEn },
  })
}
```
- Accepts an array of `{ id, valueKa, valueEn }` objects
- Updates each one individually in a loop
- Note: This is N separate queries, not a transaction. If one fails mid-way, previous updates are already committed. For a production system, wrapping in `db.$transaction()` would be safer.

### `POST /api/admin/newsletter/send`

**File:** `src/app/api/admin/newsletter/send/route.ts`

```typescript
if (preview) {
  const admin = await db.adminUser.findFirst()
  if (admin) {
    await sendNewsletterEmail({ to: admin.email, ..., unsubscribeToken: 'preview' })
  }
  return NextResponse.json({ ok: true, sent: 1 })
}
```
- Preview mode: sends only to the first admin user's email
- `unsubscribeToken: 'preview'` — the unsubscribe link in the preview email won't work (intentional)

```typescript
let sent = 0
for (const subscriber of subscribers) {
  try {
    await sendNewsletterEmail({ to: subscriber.email, ... })
    sent++
  } catch {
    // continue on individual failure
  }
}
return NextResponse.json({ ok: true, sent, total: subscribers.length })
```
- Loops through all active subscribers (not batched/queued — synchronous)
- Individual failures are silently skipped — the broadcast continues
- Returns count of successfully sent emails vs total subscribers
- ⚠️ For a list of thousands, this would time out. Consider a queue (e.g., pg-boss, BullMQ) for large broadcasts.

### `POST /api/upload`

**File:** `src/app/api/upload/route.ts`

```typescript
const formData = await req.formData()
const file = formData.get('file') as File | null
const folder = formData.get('folder') as 'blog' | 'team' | 'press' | null

if (!['blog', 'team', 'press'].includes(folder)) {
  return NextResponse.json({ error: 'Invalid folder' }, { status: 400 })
}

if (!process.env.BLOB_READ_WRITE_TOKEN) {
  return NextResponse.json({ error: 'BLOB_READ_WRITE_TOKEN is not configured...' }, { status: 500 })
}

const url = await saveUploadedFile(file, folder)
return NextResponse.json({ url })
```
- Reads `multipart/form-data` — the standard format for file uploads
- Validates that `folder` is one of the allowed values (prevents path traversal)
- Checks for the Blob token before attempting upload (gives a clear error message)
- Returns `{ url }` — the admin UI stores this URL in the relevant DB field

## NextAuth Route

### `GET/POST /api/auth/[...nextauth]`

**File:** `src/app/api/auth/[...nextauth]/route.ts`

```typescript
import { handlers } from '@/lib/auth'
export const { GET, POST } = handlers
```
- Just re-exports the handlers from the auth config
- NextAuth handles all auth-related requests at this path:
  - `POST /api/auth/callback/credentials` — processes the login form
  - `GET /api/auth/session` — returns the current session (used by client)
  - `POST /api/auth/signout` — logs out

## Consistent Error Patterns

All admin routes follow these HTTP status codes:
| Status | Meaning | When used |
|---|---|---|
| 200 | OK | Successful GET |
| 201 | Created | Successful POST (new resource) |
| 400 | Bad Request | Missing required fields, invalid ID |
| 401 | Unauthorized | No valid session |
| 404 | Not Found | Resource doesn't exist |
| 500 | Server Error | Unexpected exception |
